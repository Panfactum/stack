# Kubernetes Node Settings

**Source Code:** [Link](https://github.com/Panfactum/stack/tree/__currentPanfactumVersion__/packages/infrastructure/kube_node_settings)

**Type:** [Internal](/docs/reference/infrastructure-modules/overview)

This module is intended to setup the `user-data.toml` that Bottlerocket OS uses to launch.

The configuration settings are documented [here](https://github.com/bottlerocket-os/bottlerocket).

This should be shared across the static EKS node groups
as well as nodes provisioned via other autoscaling
mechanisms such as Karpenter.

## Providers

No requirements.

## Required Inputs

No required inputs.

## Optional Inputs

The following input variables are optional (have default values):

### <a name="input_cluster_ca_data" /> [cluster\_ca\_data](#input_cluster_ca_data)

Description: B64 encoded CA data for the cluster API server

Type: `string`

Default: `""`

### <a name="input_cluster_endpoint" /> [cluster\_endpoint](#input_cluster_endpoint)

Description: The address of the API server

Type: `string`

Default: `""`

### <a name="input_cluster_name" /> [cluster\_name](#input_cluster_name)

Description: The name of the EKS cluster

Type: `string`

Default: `""`

### <a name="input_is_spot" /> [is\_spot](#input_is_spot)

Description: Whether these settings are for a spot node

Type: `bool`

Default: `false`

### <a name="input_max_pods" /> [max\_pods](#input_max_pods)

Description: The maximum number of pods to run on this node type

Type: `number`

Default: `50`
