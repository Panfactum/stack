## Inputs

The following input variables are supported:

### allowed_vault_roles

Description: Vault roles allowed to access the Temporal UI

Type: `set(string)`

Default:

```json
["rbac-superusers", "rbac-admins", "rbac-readers", "rbac-restricted-readers"]
```

### arm_nodes_enabled

Description: Whether the database pods can be scheduled on arm64 nodes

Type: `bool`

Default: `true`

### aws_iam_ip_allow_list

Description: A list of IPs that can use the service account token to authenticate with AWS API

Type: `list(string)`

Default: `[]`

### burstable_nodes_enabled

Description: Whether the database pods can be scheduled on burstable nodes

Type: `bool`

Default: `true`

### cdn_mode_enabled

Description: Whether to enable CDN mode for the Temporal UI ingress

Type: `bool`

Default: `true`

### controller_nodes_enabled

Description: Whether to allow pods to schedule on EKS Node Group nodes (controller nodes)

Type: `bool`

Default: `false`

### default_namespace_retention_days

Description: Retention period in days for the auto-created 'default' Temporal namespace.

Type: `number`

Default: `7`

### force_search_attributes_cache_refresh_on_read

Description: Whether to force a search attributes cache refresh on every read

Type: `bool`

Default: `true`

### frontend_replicas

Description: Number of replicas. Defaults to 2 if sla_target >= 2, else 1.

Type: `number`

Default: `null`

### frontend_rps

Description: Max requests per second for the frontend service

Type: `number`

Default: `2400`

### history_replicas

Description: Number of replicas. Defaults to 2 if sla_target >= 2, else 1.

Type: `number`

Default: `null`

### ingress_domains

Description: Domains for the Temporal Web UI Ingress.

Type: `list(string)`

Default: n/a

### ingress_enabled

Description: Whether to enable the ingress for routing traffic to the Temporal Web UI

Type: `bool`

Default: `true`

### log_level

Description: Temporal server log level

Type: `string`

Default: `"info"`

### matching_replicas

Description: Number of replicas. Defaults to 2 if sla_target >= 2, else 1.

Type: `number`

Default: `null`

### max_id_length

Description: Maximum length for Temporal IDs (workflow IDs, run IDs, etc.)

Type: `number`

Default: `255`

### monitoring_enabled

Description: Whether to enable Prometheus monitoring CRs

Type: `bool`

Default: `false`

### num_history_shards

Description: Number of Temporal history shards. WARNING: This value is set once during schema initialization and can
NEVER be changed without a full data migration. 512 is appropriate for most workloads.

Type: `number`

Default: `512`

### panfactum_scheduler_enabled

Description: Use Panfactum custom scheduler

Type: `bool`

Default: `true`

### pg_initial_storage_gb

Description: How many GB to initially allocate for persistent storage (will grow automatically as needed). Can only be
set on cluster creation.

Type: `number`

Default: `10`

### pg_instances

Description: Number of DB replicas (use sla_target for default)

Type: `number`

Default: `2`

### pg_maximum_cpu_millicores

Description: The maximum amount of cpu to allocate to the postgres pods (in millicores)

Type: `number`

Default: `10000`

### pg_maximum_memory_mb

Description: The maximum amount of memory to allocate to the postgres pods (in MB)

Type: `number`

Default: `128000`

### pg_minimum_cpu_millicores

Description: The minimum amount of cpu to allocate to the postgres pods (in millicores)

Type: `number`

Default: `50`

### pg_minimum_memory_mb

Description: The minimum amount of memory to allocate to the postgres pods (in MB)

Type: `number`

Default: `500`

### pg_storage_limit_gb

Description: The maximum number of gigabytes of storage to provision for the PostgreSQL database

Type: `number`

Default: `null`

### pull_through_cache_enabled

Description: Whether to use ECR pull-through cache for images

Type: `bool`

Default: `true`

### spot_nodes_enabled

Description: Whether the database pods can be scheduled on spot nodes

Type: `bool`

Default: `true`

### temporal_version

Description: The version tag for the temporalio/server and temporalio/admin-tools images

Type: `string`

Default: `"1.26.2"`

### vault_domain

Description: The domain of the Vault instance to use for OIDC authentication on the Temporal UI

Type: `string`

Default: n/a

### voluntary_disruption_window_enabled

Description: Whether to confine voluntary disruptions of pods in this module to specific time windows

Type: `bool`

Default: `false`

### voluntary_disruption_window_seconds

Description: The length of the disruption window in seconds

Type: `number`

Default: `900`

### voluntary_disruptions_enabled

Description: Whether to enable voluntary disruptions of pods in this module

Type: `bool`

Default: `true`

### vpa_enabled

Description: Whether to create VPA resources

Type: `bool`

Default: `true`

### worker_replicas

Description: Number of replicas. Defaults to 2 if sla_target >= 2, else 1.

Type: `number`

Default: `null`

## Outputs

The following outputs are exported:

### db_admin_role

Description: The Vault role for obtaining admin database credentials

### db_reader_role

Description: The Vault role for obtaining read-only database credentials

### frontend_host

Description: The full internal hostname for the Temporal frontend (usable within the cluster)

### frontend_port

Description: The gRPC port for the Temporal frontend service

### frontend_service_name

Description: The Kubernetes Service name for the Temporal frontend (gRPC endpoint for clients)

### namespace

Description: The Kubernetes namespace where Temporal is deployed

### superuser_creds_secret

Description: The name of the Kubernetes Secret containing the PostgreSQL superuser credentials used by Temporal

### ui_port

Description: The HTTP port for the Temporal Web UI service

### ui_service_name

Description: The Kubernetes Service name for the Temporal Web UI
